Openpcdet跑通自建数据集

Openpcdet环境部署流程

本人使用硬件与环境：

Wsl:Ubuntu20.04
显卡：1650TI
CUDA 11.7 + PyTorch 1.13.1

配置步骤：

1.Conda创建虚拟环境
conda create -n openpcdet python=3.8

2.激活进入虚拟环境
conda activate openpcdet

3.安装cuda
以下命令以 ubuntu2004 为例，请务必替换为你自己的版本！
wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2004/x86_64/cuda-keyring_1.1-1_all.deb
sudo dpkg -i cuda-keyring_1.1-1_all.deb
sudo apt-get update
sudo apt-get -y install cuda-11-7

4.设置 CUDA 环境变量
echo ‘export CUDA_HOME=/usr/local/cuda-11.7’ >> ~/.bashrc
echo ‘export PATH=$CUDA_HOME/bin:$PATH’ >> ~/.bashrc
echo ‘export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH’ >> ~/.bashrc
source ~/.bashrc

5.安装 cuDNN
conda search cudnn -c conda-forge | grep cudnn
#以 cudnn 8.8.0 为例，请替换为你找到的合适版本
conda install -c conda-forge cudnn=8.8.0 -y

6.安装 PyTorch
conda install pytorch==1.13.1 torchvision==0.14.1 torchaudio==0.13.1 pytorch-cuda=11.7 -c pytorch -c nvidia -y
#验证 PyTorch 和 CUDA
python -c “import torch; print(torch.version); print(torch.cuda.is_available())”
应输出 1.13.1 和 True

7.安装 spconv
pip install spconv-cu117 # CUDA 11.7 对应 spconv-cu117

8.克隆并安装 OpenPCDet
git clone https://github.com/open-mmlab/OpenPCDet.git
cd OpenPCDet
打开requirements.txt 4，10，11需要注释掉避免冲突，再执行pip install -r requirements.txt

9.检查 gcc 版本
gcc –version
如果版本高于10，可以临时设置使用 gcc-9 进行编译（需要先安装 sudo apt install gcc-9 g++-9 -y）。

10.编译安装OpenPCDet
python setup.py develop

11.安装open3d
pip install open3d

12.下载iris数据集
https://centreborelli.github.io/iris-vis/
数据集点云密度太大用labelCloud标注软件打不开（Windows版labelCloud比较容易安装），需要下采样（降低点云密度）Cloudcompare或者python脚本都可以，然后开始标注

13.创建自定义数据集文件
cd ~/OpenPCDet
mkdir -p data/custom/{points,labels,ImageSets}
点云文件放到：points中
标注文件放到：labels中

14.创建数据集划分文件
cd ~/OpenPCDet/data/custom
echo “zoneA_clean_for_label” > ImageSets/train.txt # zoneA_clean_for_label换成自己的文件
echo “zoneB_clean_for_label” > ImageSets/val.txt # zoneB_clean_for_label换成自己的

15.创建数据集配置文件
tools/cfgs/dataset_configs/custom_dataset.yaml 需要改成自己标注的设备，点云范围

16.编辑模型配置文件
打开cfgs/custom_models/pointpillar_custom.yaml
配置文件中的点云范围要把所有数据的范围都包括，用ai生成python脚本计算每个点云文件的实际范围，custom_dataset.yaml也是一样

体素大小也许要改，不然报错，让ai告诉你应该填啥，自己算也行，PointPillar要求Z方向只有1个体素。你需要确保VOXEL_SIZE（Z方向体素大小）大于等于Z轴范围的高度。

17.将所有.ply转换为带色彩信息的.npy文件

18.生成数据信息
每次改custom_dataset.yaml这个都需删除之前的数据信息，重新生成
cd ~/OpenPCDet
rm -f data/custom/*.pkl
rm -rf ~/OpenPCDet/output/custom_models/pointpillar_custom/
python -m pcdet.datasets.custom.custom_dataset create_custom_infos tools/cfgs/dataset_configs/custom_dataset.yaml

19.开始训练
cd ~/OpenPCDet/tools
python train.py –cfg_file cfgs/custom_models/pointpillar_custom.yaml

20.运行验证程序
cd ~/OpenPCDet/tools
python demo.py –cfg_file cfgs/custom_models/pointpillar_custom.yaml
–ckpt /home/sxq/OpenPCDet/output/custom_models/pointpillar_custom/default/ckpt/checkpoint_epoch_50.pth
–data_path /home/sxq/OpenPCDet/data/custom/points/zoneA_clean_for_label.npy
–ext .npy

cd /home/OpenPCDet/tools

python demo.py
–cfg_file cfgs/custom_models/pointpillar_custom.yaml
–ckpt /home/OpenPCDet/output/custom_models/pointpillar_custom/default/ckpt/checkpoint_epoch_100.pth
–data_path /home/OpenPCDet/data/custom/points/
–ext .npy
–output_dir /home/OpenPCDet/output/demo_results

评估模型
cd /home/OpenPCDet/tools

python test.py
–cfg_file cfgs/custom_models/pointpillar_custom.yaml
–ckpt output/custom_models/pointpillar_custom/default/ckpt/checkpoint_epoch_100.pth
–eval_all
报错的话:
KeyError: ‘Redpillar’
vi /home/OpenPCDet/pcdet/datasets/kitti/kitti_object_eval_python/eval.py

服务器租赁版

清除cuda路径
unset CUDA_HOME
unset CUDA_PATH
正确设置
export CUDA_HOME=/usr/local/cuda-11.8
export PATH=$CUDA_HOME/bin:$PATH
export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH
#检查 nvvm 路径
ls -la /usr/local/cuda-11.8/nvvm/lib64/
永久保存
echo ‘export CUDA_HOME=/usr/local/cuda-11.8’ >> ~/.bashrc
echo ‘export PATH=$CUDA_HOME/bin:$PATH’ >> ~/.bashrc
echo ‘export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH’ >> ~/.bashrc
source ~/.bashrc
如果遇到cuda.h找不到的问题
export CUDA_HOME=/usr/local/cuda-11.8
export PATH=$CUDA_HOME/bin:$PATH
export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH
